# Spotd — Weekly Giveaway & Referral Program

Short context doc for the giveaway/referral system. Last updated 2026-04-29.

## What it is

Every Monday 9 AM PT, one Spotd member wins a **$25 gift card to any restaurant or bar listed on Spotd** (winner picks the venue). If the venue doesn't sell digital gift cards, the winner can swap for a digital DoorDash or Visa card.

Members earn entries by being active during the ISO week (Mon 00:00 PT → Sun 23:59 PT). Referrals stack the odds.

## How entries are earned

| Action                                    | Entries                                                    |
|-------------------------------------------|------------------------------------------------------------|
| Check in at any venue                     | 1 self entry per week (capped — doing it twice ≠ 2 entries)|
| Leave a review                            | Same self entry — stacks with check-in but doesn't double  |
| Share a photo/video in social             | Same self entry                                            |
| A user you referred is active that week   | +1 referral_bonus to **you** per active referee per week   |

A referrer can earn unlimited weekly bonuses — one per active referee per week. No fixed cap, just the natural cap of how many people they convert.

## Referral codes

- Every profile gets a unique 6-char code (no `0/O/1/I`) auto-generated by a DB trigger.
- Three ways a code gets applied to a new user:
  1. Click `https://spotd.biz/?ref=CODE` — captured on `DOMContentLoaded` into sessionStorage, applied right after signup.
  2. Type the code into the optional referral field on the signup form.
  3. Type it into the post-signup "Did someone refer you?" modal (one-time prompt).
  4. Tap "Got referred? Add a code →" link inside the giveaway tile on the profile page (fallback for late-comers).
- Self-referral is blocked at the DB level. A user can only have one referrer (`referrals.referee_id` is UNIQUE).

## Signup attribution (separate from referrals)

Onboarding screen 4 asks "How did you hear about us?" with 13 source chips: Instagram, TikTok, X/Twitter, Facebook, Google search, Email/Newsletter, Reddit, Podcast, Press, In-person event, App Store, A cool friend, Somewhere else. Selection is stashed in sessionStorage and persisted to `signup_attributions` after signup completes (covers email + Google OAuth paths). Skip is allowed.

## Database schema

```
profiles                       (existing, +referred_by uuid added)
referral_codes                 user_id PK, code unique
referrals                      referrer_id, referee_id (UNIQUE), code_used
giveaway_entries               user_id, week_start, entry_type ('self'|'referral_bonus'), source_referee_id
                               unique partial: (user_id, week_start) WHERE 'self'
                               unique partial: (user_id, week_start, source_referee_id) WHERE 'referral_bonus'
giveaway_winners               week_start UNIQUE, winner_user_id, total_entries, winner_entry_count, prize_status
signup_attributions            user_id PK, source, source_other
```

Triggers fire `grant_giveaway_entries(user_id, action_at)` on insert into `check_ins`, `reviews`, `checkin_photos` (the photo/video posts in the social section — not the admin `social_posts` content-calendar table). The function inserts the user's self entry + the referrer's bonus entry, both idempotent via partial unique indexes.

Helper functions: `current_week_start_pt()`, `week_start_pt_for(ts)`, `is_giveaway_admin()` (currently allows `shanerutter@gmail.com` only).

## Admin RPCs (all gated by `is_giveaway_admin()`)

- `admin_giveaway_leaderboard(p_week_start date)` — every user with entries that week, with self/referral counts and email
- `admin_record_winner(p_week_start, p_winner_user_id, p_prize_notes)` — atomic winner write
- `admin_list_winners(p_limit)` — past winners with status
- `admin_set_prize_status(p_week_start, p_status, p_prize_notes)` — pending → sent → delivered
- `top_referrers_30d()` — leaderboard
- `admin_attribution_breakdown(p_since, p_until)` — counts per source
- `admin_attribution_recent(p_limit)` — last 50 attributed signups
- `admin_attribution_coverage(p_days)` — % of recent signups that have a source recorded

## Weekly admin workflow (Monday)

1. Open admin portal → **🎁 Giveaway** tab in the sidebar
2. Click **Last week**
3. Either click **🎲 Pick random winner** (weighted random by ticket count), or click **Choose as winner** on a specific row
4. The user's email becomes a clickable mailto with a pre-filled "You won" subject line — write the message, ask which venue they want the $25 gift card from
5. Buy the digital gift card at the venue's site (or DoorDash/Visa if they swap)
6. Click **Mark sent** on their row in Past Winners
7. When they confirm receipt, click **Mark delivered**

Bonus tab: **📊 Attribution** for the last 7/30/90d signup-source breakdown.

## Security posture

- All admin RPCs are `SECURITY DEFINER` with pinned `search_path` and gated by `is_giveaway_admin()`.
- `grant_giveaway_entries` execute is revoked from `anon`/`authenticated` so a logged-in user can't grant themselves entries by calling the RPC directly. Triggers run with the function owner's privileges, so they keep working.
- RLS policies: users can only read their own `giveaway_entries`; `referral_codes` is publicly readable (so signup can look up a code); `giveaway_winners` is publicly readable (so the landing page can show recent winners).

## What's not built (deliberately)

- **Automated cron pick** — the edge function and cron schedule were dropped in favor of a manual Monday workflow. The function file still exists in the repo but is unused.
- **Email notifications** — winners are emailed manually by Shane via the mailto link on the admin page. Loops transactional templates were planned but not created.
- **Fraud detection** — no IP rate-limiting, no account flagging. Acceptable at <1k users; revisit if abuse appears.

## Operational notes

- ~75 users at launch, San Diego only.
- Gift card fulfillment is manual: Shane checks the winner's pick has digital cards, buys it, emails the code.
- The DoorDash/Visa fallback is the safety net for venues without digital cards.
- Treat any winner with implausible entry counts (e.g. 50 referral bonuses) as a fraud signal. Exclude and re-pick.
