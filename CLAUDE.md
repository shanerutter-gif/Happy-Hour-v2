# Claude Code — Project Context

## Which app is live on spotd.biz

**The vanilla JS app at the repo root is live on `spotd.biz`.** The React
rewrite under `spotd-app/` is **not** currently deployed or used in
production.

When making changes in response to user requests, default to editing the
**vanilla JS app** unless the user explicitly says otherwise.

### Vanilla JS app (live — edit these)
- `index.html` — main SPA entry
- `blog.html`, `blog/*.html` — blog index and posts
- `business-landing.html`, `business-portal.html`, `business.html`
- `admin.html`, `admin/*`
- `css/*` — stylesheets
- `js/*` — client JS
- `api/*.js` — Vercel edge functions (e.g. `spots.js`, `sitemap-venues.js`,
  `daily-deals.js`, `loops-*.js`, `stripe-*.js`, `send-push.js`, `auth.js`)
- `vercel.json` — routes, crons, redirects
- `sitemap.xml`, `robots.txt`, `manifest.json`, `sw.js`
- `sql/*` — Supabase schema / migrations
- `emails/*` — transactional email templates

### React rewrite (NOT live — do not edit unless explicitly asked)
- `spotd-app/` — Vite + React 19 + React Router port. Kept in the repo
  for future migration work but not deployed to `spotd.biz`.

## Deploy

- Hosted on Vercel project `happy-hour-v2` (team
  `shanerutter-2028s-projects`).
- Production domains: `spotd.biz`, `www.spotd.biz`.
- Vercel build serves the static files at the repo root; `api/*.js` runs
  as edge functions.
- Pushes to `main` auto-deploy to production.

## Data

- Backend is Supabase. Venue pages are generated on-the-fly from the
  `venues` table by `api/spots.js` at `/spots/<name-slug>`; `active=true`
  rows appear automatically. Sitemap for venues is generated by
  `api/sitemap-venues.js` at `/sitemap-venues.xml`.
